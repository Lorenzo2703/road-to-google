## Phase 0: Pre-Work

### Application Preparation
- [ ] Read the full job description 3 times and highlight key requirements
- [ ] Research the Agent Development Kit (ADK) team at Google
- [ ] Identify which qualifications you already meet
- [ ] Identify gaps between your profile and the role requirements
- [ ] Set a target application date (ideally after 6 weeks of preparation)

> **Why it matters**: Understanding exactly what Google wants lets you focus your preparation. Every hour spent on the wrong topic is an hour wasted.

> **Completion criteria**: You can list all 5 responsibilities and 4 preferred qualifications from memory.

### CV Improvement
- [ ] Update CV to be 1 page (Google prefers concise CVs)
- [ ] Add quantifiable impact metrics to every bullet point (e.g., "reduced latency by 40%")
- [ ] Highlight experience with Python, SDKs, open-source, or AI/ML
- [ ] Add any agent/LLM related projects prominently
- [ ] Tailor the summary/objective section to this specific role
- [ ] Remove irrelevant experience or skills
- [ ] Have 2-3 people review your CV
- [ ] Use action verbs: Designed, Built, Deployed, Optimized, Led, Collaborated
- [ ] Ensure consistent formatting and zero typos
- [ ] Add a link to your GitHub profile

> **Why it matters**: Your CV is the first filter. Google recruiters spend ~15 seconds per CV. If the keywords don't match, you won't get an interview.

> **How to do it**: Use the XYZ formula: "Accomplished [X] as measured by [Y], by doing [Z]". Example: "Reduced API response time by 35% by implementing caching layer with Redis, serving 10K+ daily requests."

> **Resources**: [Google Resume Tips](https://www.techinterviewhandbook.org/resume/), [FAANG Resume Guide](https://www.careercup.com/resume)

> **Completion criteria**: CV is 1 page, has 5+ quantified achievements, and 2+ people have reviewed it.
